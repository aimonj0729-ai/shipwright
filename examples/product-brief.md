# Example: Trend To Product Brief

## Trend

Developers are sharing and installing AI agent skills because they want repeatable workflows, not blank chat boxes.

## User Pain

Indie builders see hot repositories every day but struggle to convert that signal into a focused project they can ship.

## Product Opportunity

An installable skill pack that turns GitHub trends into product opportunities, PRDs, GitHub issues, and launch checklists.

## MVP

- A `github-radar` skill that filters hot projects.
- A `trend-to-product` skill that identifies a narrower user and wedge.
- An `idea-to-prd` skill that creates a lean product brief.
- A `prd-to-issues` skill that creates a build sequence.
- A `launch-readiness` skill that audits the repo before posting.

## Differentiation

Most prompt packs stop at ideation. This package connects the full path from trend discovery to public launch.

## GitHub Launch Shape

Repo name: `shipwright`

Promise: Practical Claude Code and Codex skills for turning trends into shippable products.

Demo: Show one hot GitHub repo becoming a PRD and issue list in three prompts.

## Risks

- Too broad: keep the first version focused on builders.
- Too generic: include concrete output formats and examples.
- Too many skills: keep each skill tied to a single repeated pain.
